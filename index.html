    Get Features

    This endpoint retrieves all features.

    GET /features HTTP/1.1

    HTTP/1.1 200 OK
Content-Type: application/json
[
  {
    "id": 1,
    "strategies": [
      {
        "id": "location-strategy",
        "payload": {
          "location": "barcelona"
        }
      }
    ],
    "isEnabled": true
  },
  {
    "id": 2,
    "strategies": [],
    "isEnabled": true
  }
]

    Get Feature

    This endpoint retrieves a specific feature.

    GET /features/{feature_id} HTTP/1.1

    HTTP/1.1 200 OK
Content-Type: application/json
{
  "id": 1,
  "strategies": [
    {
      "id": "location-strategy",
      "payload": {
        "location": "barcelona"
      }
    }
  ],
  "isEnabled": true
}
    Parameter Description
    feature_id The ID of the feature to retrieve

    Post Feature

    This endpoint creates a feature.

    POST /feature/{feature_id} HTTP/1.1

    HTTP/1.1 201 Created
    Parameter Description
    feature_id The ID of the feature to create

    Patch Feature

    This endpoint patches a specific feature.

    There are four different actions:

    • Enable Feature
    • Disable Feature
    • Add Strategy
    • Remove Strategy
    PATCH /feature/{feature_id} HTTP/1.1
{
  "action": "add_strategy",
  "value": {
    "strategy_id": "my-first-strategy",
    "strategy_type": "STRATEGY_TYPE"
  }
}

    HTTP/1.1 202 Accepted
    Parameter Description
    feature_id The ID of the feature to retrieve
    Action Description
    enable_feature Enable Feature
    disable_feature Disable Feature
    add_strategy Add Strategy to Feature
    remove_strategy Remove Strategy to Feature
    Value Used in Action
    strategy_id add_strategy, remove_strategy
    strategy_type add_strategy
    Both add_strategy and remove_strategy require a `value` on the request body.
    Only add_strategy requires the strategy_type on the request body.

    Delete Feature

    This endpoint deletes a specific feature.

    DELETE /features/{feature_id} HTTP/1.1

    HTTP/1.1 204 Deleted
    Parameter Description
    feature_id The ID of the feature to delete

    Status Codes

    Toggle CRUD PSR7 API uses the following HTTP status codes:

    Code Meaning
    200 OK -- Request was successful
    201 Created -- Feature created
    202 Processed -- Feature patch processed
    204 Deleted -- Feature deleted
    400 Bad Request -- Request is invalid
    404 Not Found -- Specified feature was not found
